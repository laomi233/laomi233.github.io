    就是说要找到一个字符串里面最长的回文字串。这个题目有一个专门的算法——Manacher

    我自己的思路就非常暴力了emmm，枚举子串的头和尾，然后再判断这个子串是否是回文，时间复杂度高达O(n^3)。所以就跳过这个解法吧，毕竟Manacher时间复杂度O(n)啊！还是来看看顺便复习Manacher算法。

    下面是通过网上总结Manacher算法思路。

    举个栗子，现在有字符串s= “aba”。

    • 第一步：预处理字符串

      我们把这个字符串填充成这样 #a#b#a#，为什么要填充成这样呢？是因为我们为了避免字符串长度是偶数或者是奇数的不同操作，所以干脆不管是奇数偶数我们都把他改成奇数长度，插入的字符必须是原字符串没有出现过的，这样的话就能保证原来是回文的子串插入之后仍然是回文，不是回文的插入后也不会是回文。

      注意 ：我们还需要在字符串的开头加入一个另外的不同的字符’$’，这样的目的是为了当对某个位置匹配越界

    • 第二步：求数组p的值

      我们要建立一个数组p，p[i]表示的是，以i为中心，向左或向右拓展的最大长度使得s[i-p[i]]到s[i+p[i]]是一个以i为中心的回文字符串。那么要怎么求这个p[i]呢？

      我们先来通过s = “abc”这个例子，通过肉眼看出来p数组是啥

    那这个p[i]是怎么推出来的呢？假设已存在一个回文，回文的中性轴为id，i > id。我们已得知p[id]。那么回文的右边界最大可达到i+p[i]，把这个值我们设为mx。如果i > mx，那没办法我们只能重新开始匹配，因为我们这时候是无法通过已经得到的回文来推出的，所以p[i] = 1。如果 i <= mx，让我们来看下下面的图解

    我们可以的到i和mx分别关于id对称的下标，i’和mx’。

    • i’ = id - (i-id) = 2*id-i

    • 假设p[i’] <= mx-i

      由于i’+p[i’]和i’-p[i]在回文内，所以：a = reverse(d), b = reverse(c) , a = reverse(b) —-> c = reverse(d)

      则p[i] = p[i’]

    • 假设p[i’] > mx-i

      可以看到，这样的话只有p[i]=mx-i才能保证i为中心的字符串是回文，超过mx的部分就不能确定只能一个一个更新了。

      若i >= mx，已经超过了这个以id为中心的字符串，那么现在只能得知最大的回文串就是i下标的字符他自己

      综上所述：p[i] = min(p[i’], mx-i)（i <mx); p[i] = 1 (i >= mx)

    • 第三步：得到原字符串的最长回文子串。通过p可以发现一个神奇的结论，再来看一下这个表：

      可以知道这个原来字符串的最长回文子串长度是3。为什么是3呢？p的最大值是4，也就是说在b这个字符对折之后的回文串长度是4，但是由于在字符串的每个字符前后都插入一个#，那么可以知道这个回文串原来的长度是(p[i]/2-1)+1+(p[i]/2-1) = p[i]-1。所以原字符串最长的回文串长度是3。

    • 讨论时间复杂度的问题：为什么这个算法的时间复杂度是O(n)?

      我们可以看到每次进入for循环的p[i]有三种选择。

      1. p[i]取p[2*id-i]。这个时候我们可以知道p[2*id-1]是比mx-i要小的，若i加上一个比p[2*id-i]的数字后仍是一个回文字符串，那么由于对称性，p[2*id-i]的值应该更大才对。所以这个取值的时候，并不会进入for里面的while
      2. 当p[i]取值mx-i, 若我们继续进入while，且p[mx+1]!=p[2*mx-i-1]，那么和上面一样就不会进入while循环了，并且mx的值保持不变。如果条件可以进入while循环，那么扩展结束后mx变大，假设扩张前mx的值是a[k],扩张后变为a[k+1], 可以知道while循环总共执行了a[k+1]-a[k]次(i <= a[k] <=a[k+1])
      3. 若i>=mx, 则会进入while循环，且结束后mx的值会变成大于等于i，假设扩张前mx的值是a[k],扩张后变为a[k+1]，则while循环执行的次数是a[k+1]-i。(a[k] <= i <=a[k+1])
      4. 综合上面三种情况，while循环每次执行最多a[k+1]-a[k]次。如果每个字符我们都进行了扩展，那么while执行的总次数是a[1],a[2]-a[1],……,a[n]-a[n-1]，他们的和是a[n],a[n]是mx的值，是小于预处理后字符的长度的，假设预处理后的字符长度是n，那么整个时间复杂度就是O(n)的，若n是原字符串长度，时间复杂度是O(2*n+1)。
