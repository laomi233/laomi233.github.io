    中文意思：求两个数相除之后的商，不能用乘法，除法，模运算。（注意溢出处理）

    开始的思路： 被除数不断减去除数，直到被除数比除数小为止。run后符合预期的TLE。

    改进的思路 ：由于一直将被除数递减，如果被除数达到int的最小值，将会产生溢出问题。而且这种累加的方法来得出答案非常慢，若divisor是1，时间复杂度达到O(n)。考虑把累加接近被除数的方法改成将除数倍乘来接近被除数的方法，从而降低时间复杂度。当找到一个最接近的数字时，便将被除数减去这个数字继续执行之前的代码。

    举个栗子：10 / 3 = 3……3。 由于我们不能使用乘法，所以我们通过二进制中左移一位和乘2等同的办法，将除数不断乘2来找到一个最接近被除数的数字。比如在这个例子中，我们设ans为要求的商 ，3 << 1 = 6, 6 << 1 =12，于是我们可以通过不断乘2来的到这样最接近被除数的数字为6，于是我们得知现在我们已经得到了两倍的3，因为（1<<1) = 2，ans累加2，我们把被除数减去现在的6，还剩4，依然比除数大，所以我们继续以上相同的步骤，将4不断乘2倍来接近被除数，由于 3 << 0 = 3是最接近4的数，（1<<0) = 1, ans累加1。被除数4减去3为1，现在已经比除数小了，因此循环停止，ans为3；可以看到该算法的时间复杂度最高为（log(n)*log(n))
