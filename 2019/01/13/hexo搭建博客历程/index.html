    hexo搭建博客（折腾）历程

    打一开始搭博客，我是没有想到会踩到这么多坑的。。。

    首先：

    • 采用jekyll + github page 搭建博客。使用这个框架需要下载ruby，下载完之后在bundle install那里卡了很久，谷歌百度都搜过了发现是需要翻墙的问题，国内的镜像速度也非常慢。于是放弃┑(￣Д ￣)┍
    • 转向用hexo + github page搭博客。果然用这个框架搭建很快，网上的教程也十分详细，基本上找到一篇像样的博客然后跟着一步一步走就搭好啦。
    • 三个阶段：
      • hexo + github page 搭建框架
      • 绑定自定义域名
      • 更换主题

    更换主题

    ​ 没想到在更换主题这里踩了很久的坑，主要是自己只能看懂一点js代码对node.js完全不懂让自己走了好久弯路（我只是想搭个博客好好写博文而已啊இ௰இ）。主要是配置文件那里，后面会讲怎么解决。

    ​ 当然如果你觉得默认主题已经很好看了就跳过这部分专心写博客吧。但是作为一个致力于折腾的人。这一步我是不会放弃的。

    • 请先挑选一个你喜欢的主题，你可以到github或者google hexo themes，到第一个网站去挑选。

    • 我挑选的主题github地址：https://github.com/Ben02/hexo-theme-Anatole

    • 查看readme会有详细的安装中文文档，跟着做就ok。

    • 按照文档安装过程中我踩到的一个坑是顶部导航关于和友链两个页面的连接，如果自己没有新建这两个页面点击之后是404。issue里面也有关于这个问题的解决方案。奈何自己对前端知识所知尚浅，在自己本地文件夹的命令行hexo new page “about”之后虽然已经新建了about这个文件夹，但是还少了一步：

      到_config.yml文件的底部添加

      menu:

      About: /about

      Links: /links

      关于配置文件还有一个要改动的地方，进入自己本地文件夹的_config.yml。注意不是themes文件夹里面自己的主题的文件夹，然后把deploy那里的内容修改如下：repository是你的仓库的ssh地址
