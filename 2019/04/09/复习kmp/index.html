    • 对应①，i和j都为0，这时候p[0] != t[0]，暴力解决的方法是i向后挪一位，p整个字符串也向后挪一位，到达②。

    • 按照①的做法，我们也只是将整个字符串p向后挪一位再去和t匹配。从而到达⑥

    • 可是很显然，上一步的操作要让i回溯到5，j要从头开始为0，也就是这里提高了复杂度。而kmp就是为了解决i回溯还要j不要又从0开始的问题，使得i不动或者加一，并且通过预处理的方法知道j要回到哪里。

      在不知道kmp是如何实现的情况下，我们先来看下kmp遇到上面⑤的情况时，是怎么表现的

    • 回到⑤：此时i = 4，j = 0。我们终于找到了p的首字符第一次在t中匹配的地方。于是我们不挪动p，让j++看下一个字符与t[i++]是否匹配。当j加到6时,i = 10,p[j] = D, t[i] = ‘ ‘。此时p和t失配。当前i = 10，j = 6。

      此时的做法是：让j回到2，让i不动仍为10，查看p[j]和t[i]是否相等，也就是说让p向后挪4个位置再继续和t匹配

      为什么这样做？

      因为我们知道匹配的字符有ABCDAB，这里刚好能知道我们有一个前缀AB和一个后缀AB是相同的，相同的元素个数有2。那么根据对称，前缀AB挪4位到后缀AB的位置时，它也会跟t中对应的元素匹配。所以我们能够找到一种更快更好的移动方式将p再次进行和t匹配而不使i回溯到i-j+1的位置，而j只是回到跟后缀相同的最长前缀的下一个位置。从而放弃了将j又从0开始，i回溯到i-j+1的位置对比较过的元素又继续进行比较的时间复杂度较高的方法。
